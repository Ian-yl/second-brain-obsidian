# 引导用户快速建立【自己的】知识体系

> **给 LLM / agent 的执行说明**：照这些步骤，把用户刚建好的空第二大脑，快速填成有【用户自己】知识的体系。写入 vault 的格式一律照 `references/vault-format.md`。
>
> **⚠️ v2 修正（2026-07-01）**：v1 把方法论（费曼 / 切片…）当"种子内容"导入用户库——**错了**。**方法论是 agent 建知识的【手法·技能】，不是用户的知识储备**；用户库只装用户自己的知识（决策 / 经验 / 项目 / 在学的）。v2 把 bootstrap 聚焦到**快速把【用户的】知识灌进来**（费曼抽取 + 导入用户资料），方法论回归 agent 手法层、种子包降级为"可选参考"。

---

## 0. 铁律（把"软提示被 agent 跳过"的坑写死）

之前踩过的坑（采访没先问语音、只提 Azure 没提 MiniMax、没帮装 Obsidian）**根子都是"重要动作被写成软备注 / 可选 / `>` 提示，LLM 忙起来就跳过"**。本方案所有重要动作**一律写成硬步骤**：

1. **标 `【必做】` / `【必问·等答】` 的步骤 = 硬性**：必须显式做、等用户回答再继续；绝不写成软备注、绝不默认跳过、绝不闷头进下一步。
2. **每次写库后 = 必留页脚** `🧠 第二大脑：<记了啥 → 路径>`，让用户每步看得见。
3. **"可选"只用于用户的选择，不用于 agent 的动作**：offer（问不问）必做；用户选不选才可选。
4. **别默认帮用户做决定**：给选项、等答；没回应就停在这步。

## 1. 目标 & 问题

- **目标**：引导用户快速建立【自己的】知识体系——建库后快速有**用户自己的**知识，不再空骨架。
- **问题**：库建好只有「结构 + 画像」，知识全靠慢速「在场提炼」一条条长；用户建完面对空库、没成就感。
- **关键（v2）**：填库的是**用户自己的知识**（费曼抽取 + 导入他的资料），**不是通用方法论**——方法论是 agent 用来建知识的手法。

## 2. 核心原则：方法论是 agent 的手法，不是用户内容

- **费曼** = **提取**手法：让用户"用大白话讲出来"→ 捕获真懂的 + 暴露盲区。用于采访 / 抽取。
- **切片** = **组织**手法：把用户知识切成**原子笔记**（一念一片、`[[]]` 互链）。决定笔记粒度。
- 其余（卡片盒 / 间隔重复 / PARA / 第一性 / 复盘）= 组织 / 思考框架。
- 这些手法在 `references/methodologies/`，agent 读它、**把手法用在用户的知识上**；**默认不当内容塞进用户库**（用户库 = 数字化的用户，通用学习法不属于这里）。

## 3. 触发（硬闸门）

- **【必做 · 硬闸门】建库 + 采访结束，立刻问知识起步，别直接进日常**：
  > 「第二大脑建好、画像也写好了 🎉。库里还没**你的**知识——要不要现在快速灌进来？① 我用**费曼法**帮你把你懂的 / 在学的记下来；② **喂我你的**链接 / 书单 / 笔记，我导入。（先不用也行）」
  → 等用户选；"先不用"才进日常。
- **随时**：「导入知识 / 记录我懂的 / 喂资料 / 费曼记一下」→ 进对应模式。

## 4. 模式（都写成硬步骤）

### ① 费曼抽取（主推 · 把用户脑子里的搬进来）
1. **【必问·等答】**「你最懂 / 在学的 3-5 个领域 / 主题是啥？」
2. 对每个用**费曼法**（`methodologies/feynman.md`）：问「用大白话讲讲你懂的 <X>？」→ 讲清的写成笔记（`sources: 用户明说`）；含糊 / 跳过的 → 标 `> 待补：<盲区>`（`sources: 推断·待确认`，别当事实）。
3. 大主题用**切片法**（`methodologies/atomic-slicing.md`）切成原子笔记 + `[[]]` 互链；按 vault-format 判位入库。
4. **页脚**：`🧠 第二大脑：记了 <X> + N 个待补盲区 → <路径>`。

### ② 批量导入用户资料（把存量搬进来）
- **URL / 书单 / 课程**：用户给一串 → 【逐条】WebFetch 抓正文（书 / 课无正文 → 问"你记得的核心要点"）→ 按 vault-format 判位入库（判层 + 模式 + 去重）。
- **文件夹 / 现有笔记**：用户指路径 → 读 → 按 vault-format 重构入库（内容多 → **切片法**拆原子笔记）；原文件不动。

### ③ 类型模板
建知识笔记按类型套 `references/note-templates.md`（概念 / 决策 / 复盘 / 读书笔记 / 项目）——结构统一、可检索。

### （可选）学习法参考 —— 默认不推
用户**明确说想要**一份学习法速查当参考 → 才读 `methodologies/<id>.md` 导入 `70-Assets/参考/`、标明「参考资料·非我的知识」。**默认不主动灌**——方法论是 agent 的手法，不是用户的知识储备。

## 5. 方法论作为 agent 手法（怎么"用"而非"存"）

- **采访（§B）本质 = 费曼法**：让用户把「你是谁」讲清，讲不清处就是盲区。
- **提炼 / 建笔记 = 切片法**：一念一片、原子粒度、`[[]]` 互链。
- agent 遇到具体建知识场景就翻 `methodologies/<id>.md` 应用对应手法；**不把手法本身写进用户库**。

## 6. 反跳过检查表

- [ ] 建库+采访后的"知识起步"offer 是**硬闸门**（§3）——SKILL §B 收尾处**必做**、不是 `>` 备注 / 不是"可选"。
- [ ] 每个模式动作是**编号硬步骤**、`【必问·等答】` 处等答。
- [ ] 每次写库**必留 `🧠` 页脚**。
- [ ] 入库的是**用户自己的知识**（`sources: 用户明说` / 推断标 `待确认`）；**方法论不默认进用户库**（除非用户明确要参考 → `70-Assets/参考/`、标明非用户知识）。
- [ ] 知识笔记 `aliases` 必含标题（防 MOC / 00-Home 断链）；`相关:` 只留真存在的链接。
- [ ] 全程 agent 直接读写 markdown、**零新依赖**；路径跨平台、**别用 `/tmp`**。

## 7. 分期（已实现）

1. **Phase 1**：模式①费曼抽取 + 手法整合（采访=费曼、提炼=切片）+ SKILL `## 知识导入` 一节 + 建库收尾硬闸门。
2. **Phase 2**：模式②批量导入（URL / 书单 / 文件夹）。
3. **Phase 3**：模式③类型模板（`references/note-templates.md`）。
- **方法论种子包**（v1 的 Phase 1）→ **降级为"可选学习法参考"**（默认不推）；`references/methodologies/` 保留为 **agent 手法工具箱**。

## 8. 验收（怎么算做到了）

- 新用户建库 → **必被问**要不要灌【你的】知识（不静默进日常）；
- 选费曼抽取 → 库里出现**用户自己**的知识笔记（`sources: 用户明说`）+ 盲区标注；
- **方法论【不】默认出现在用户库**（除非用户明确要参考 → 落 `70-Assets/参考/`、标明非用户知识）；
- 结构校验（`tests/build_validate.py` 式）：新笔记 frontmatter / tags / aliases / 链接全过。
