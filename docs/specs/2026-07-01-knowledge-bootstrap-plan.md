# 知识快速导入 · 引导用户快速建立知识体系

> **给 LLM / agent 的执行说明**：这是一份**可执行方案**——agent 照这些步骤，把用户刚建好的空第二大脑，快速填成"有干货的知识体系"。
> **本文件是"要建什么"的规格；写入 vault 的格式一律照 `references/vault-format.md`（去重 / 判位 / frontmatter / aliases / 同步索引都在那）。**

---

## 0. 铁律（吸取教训 · 把"软提示被 agent 跳过"的坑写死）

之前踩过的坑（采访没先问语音、只提 Azure 没提 MiniMax、没帮装 Obsidian）**根子都是"重要动作被写成软备注 / 可选 / `>` 提示，LLM 忙起来就跳过"**。本方案所有重要动作**一律写成硬步骤**，且遵守：

1. **标 `【必做】` / `【必问·等答】` 的步骤 = 硬性**：**必须显式做、等用户回答再继续**；**绝不写成软备注、绝不默认跳过、绝不闷头进下一步、绝不"一段话糊过去"**。
2. **每次写库后 = 必留页脚**：`🧠 第二大脑：<记了 N 篇 → 路径>`（同 SKILL §提炼 的页脚仪式），让用户每步看得见。
3. **"可选"只用于用户的选择，不用于 agent 的动作**：offer（问不问）是**必做**的；用户选不选、选哪个，才是可选。
4. **别默认帮用户做决定**：给选项、等答；用户没回应就停在这一步，别擅自继续。

---

## 1. 目标 & 问题

- **目标**：**引导用户快速建立起知识体系**——建库后不再是空骨架、立刻有可用的知识 + 方法论。
- **问题**：现在库建好只有「六大工作模式结构 + 画像」，知识全靠慢速的「每轮在场提炼」一条条长；用户建完面对空库、没即时可用的东西、没成就感。

## 2. 核心原则：方法论有「双重身份」

费曼、切片这些**不只是要存的内容，更是 agent"怎么帮用户建知识"的操作手法**：
- **费曼法** = **提取技术**：让用户"用大白话讲一遍" → 逼出真懂的 + 暴露盲区 → 这就是采访 / 提炼的核心手法。
- **切片法** = **组织技术**：把大知识切成**原子笔记**（一念一片、用 `[[]]` 互链）→ 这就是知识笔记该有的粒度。

→ 所以本方案把学习法**既当"内容种子"、又当 agent"建构手法"**。

---

## 3. 触发时机（硬闸门）

- **【必做 · 硬闸门】建库 + 采访（§A+§B）一结束，立刻问知识起步，别直接进日常**：
  > 「你的第二大脑建好、画像也写好了 🎉。现在库里还没知识——要不要**快速灌点东西打底**？三种方式（选一个 / 说都要 / 说先不用都行）：
  > **① 方法论种子包** —— 我灌一套学习·思考方法论（费曼、切片、间隔重复…），马上有干货、也顺带教你怎么用第二大脑；
  > **② 喂链接 / 书单** —— 你给我几个链接或书名，我抓来提炼成笔记；
  > **③ 费曼法帮你记** —— 挑一个你懂 / 在学的领域，我用"讲大白话"的方式帮你快速记下来。」
  → **等用户选**再进对应模式；用户说"先不用" → 才进日常（§提炼）。
- **随时触发**：用户说「导入知识 / 灌方法论 / 快速建知识库 / 费曼记一下 / 切片整理」→ 进对应模式（4/5/6 节）。

## 4. 模式 ① 方法论种子包（Phase 1 · 先做这个）

### 4.1 种子清单（skill 自带，放 `references/methodologies/<id>.md`）
| id | 方法 | 一句话 |
|---|---|---|
| `feynman` | 费曼学习法 | 用大白话把概念教出来，讲不清的就是没真懂 |
| `atomic-slicing` | 切片 / 原子笔记 | 一念一片、互相链接，让知识长成网 |
| `zettelkasten` | 卡片盒笔记法 | 永久笔记 + 唯一 ID + 链接，卢曼式知识增殖 |
| `spaced-repetition` | 间隔重复 | 按遗忘曲线复习，把知识钉进长期记忆 |
| `para` | PARA | Projects / Areas / Resources / Archives 组织信息 |
| `first-principles` | 第一性原理 | 拆到不可再拆的基本事实，从头推 |
| `retrospective` | 复盘法 | 目标→结果→差距→归因→改进（PDCA / AAR） |

### 4.2 执行步骤（【必做】按序，别跳）
1. **【必问·等答】列清单 + 问范围**：把上面清单给用户看，问「要**全部**、还是**推荐 5 个（费曼 / 切片 / 间隔重复 / PARA / 复盘）**、还是**你自选几个**？」——等答。
2. **逐篇写入**：对每个选中的 id → 读 `references/methodologies/<id>.md` → 按 `vault-format.md` 写进 `20-Process/学习成长/<YYYY-MM-DD> - <方法名>.md`。frontmatter 必含：`type: knowledge`、`layer: Process`、`mode: learning-growth`、`tags: [类型/方法论, 层/Process, 模式/学习成长]`、`aliases: [<方法名>]`（否则 MOC 里 `[[方法名]]` 链不过去——见 vault-format）。**种子笔记的 `相关:` 行会互相引用——只保留指向【本次也导入了】的方法论，指向没导入的那些链接删掉（否则断链）；建议一次导全 7 篇、互链最完整。**
3. **建 / 更 MOC**：更新 `50-MOCs/MOC - 学习成长.md` 的 `## Process` 段，逐条加 `- [[<方法名>]]`。
4. **更新 `00-Home.md`**（同 vault-format §4）。
5. **【必做】页脚留痕**：`🧠 第二大脑：导入 N 篇方法论 → 20-Process/学习成长/`。
6. **收尾一句引导**：「灌好了，随时可以用它们——比如说『用费曼法帮我记 X』，或直接聊、我在场提炼。」

### 4.3 每篇种子笔记的正文结构（固定 5 段）


### 4.4 种子内容 · 费曼学习法（`references/methodologies/feynman.md` 正文）
```
# 费曼学习法

## 是什么
通过「把概念用大白话教给一个小白」来学习——**讲不清楚的地方，就是你没真正懂的地方**。

## 怎么用（步骤）
1. 选一个你想搞懂的概念，写在纸/笔记顶端。
2. 假装教给一个完全不懂的人，**只用大白话、不用术语**讲一遍。
3. 卡壳、含糊、要靠术语糊弄的地方 → **标出来，那就是盲区**，回原始资料补。
4. 补完再讲一遍，精简到顺畅、连小孩都听懂为止。

## 什么时候用
学新概念、检验"我是不是真懂"、准备讲解 / 写作 / 面试前。

## 一个例子
学"复利"：别背公式，试着跟朋友说「钱生的利息，下一轮也跟着生利息，滚雪球」——如果你能顺畅打这个比方，你就懂了；如果只会念公式，说明还没懂。

相关：[[切片法]] [[卡片盒笔记法]]
```

### 4.5 种子内容 · 切片法 / 原子笔记（`references/methodologies/atomic-slicing.md` 正文）
```
# 切片法 / 原子笔记

## 是什么
把大块知识**切成"一念一片"的原子笔记**——每篇只讲一个概念 / 结论，能独立看懂；再用 `[[链接]]` 把相关的连成网。

## 怎么用（步骤）
1. 遇到一个大主题，先**拆成若干独立小点**（每个小点能一句话说清）。
2. 每个小点写成**一篇短笔记**（标题=这个点、正文几句话，脱离上下文也读得懂）。
3. 用 `[[]]` 把相关笔记**连起来**（"这个让我想到那个"）。
4. 别追求一次写全——让知识网**随用随长**。

## 什么时候用
整理复杂主题、建可复用的知识网、避免笔记变成"一大坨堆料、以后找不到"。

## 一个例子
学"机器学习"别写一篇万字长文；切成：`[[什么是过拟合]]`、`[[训练集vs测试集]]`、`[[梯度下降一句话]]`… 每篇独立、互链，下次单独复用某一片很方便。

相关：[[费曼学习法]] [[卡片盒笔记法]]
```
> 其余 5 篇（卡片盒 / 间隔重复 / PARA / 第一性原理 / 复盘法）Phase 1 实现时照 4.3 的结构补齐。

## 5. 模式 ② 批量导入用户已有知识

按用户给的来源分支，每条都按 `vault-format.md` 判位入库、页脚留痕：
- **喂 URL / 书单 / 课程名**：用户给一串 → 【逐条】WebFetch 抓正文（书 / 课无正文则问用户"你记得的核心要点是什么"）→ 提炼成笔记（判层 + 模式 + 去重）。
- **导入现有笔记 / 文档**：用户指一个文件夹 / 文件 → 读 → 按 vault-format 重构入库（一篇拆多篇原子笔记时用**切片法**）→ 原文件不动、只入库提炼版。
- **一次性"知识访谈"**：**【必问·等答】**「你最懂 / 在学的 3-5 个领域是什么？」→ 对每个用**费曼法**（模式③）逼用户讲两句 → 秒变笔记。

## 6. 模式 ③ 费曼 + 切片驱动的快速抽取

- **费曼式抽取**：`【必问】`「用大白话给我讲讲你懂的 <主题>？」→ 用户讲 → agent：① 把讲清的部分写成笔记（`sources: 用户明说`）② 讲含糊 / 跳过的 → 标成笔记里的 `> 待补：<盲区>`（`sources: 推断·待确认`，别当事实）③ 页脚报记了啥 + 几个待补盲区。
- **切片式抽取**：用户丢一个大主题 → agent 帮**切成原子笔记**（每片一句话说清）+ 互链 → 逐片确认后入库。

## 7. 模式 ④ 知识类型轻模板

导入时按类型套模板（放 `references/methodologies/_templates.md` 或 `_System/Templates/`），让结构统一、可检索：
- **方法论**（见 4.3）、**概念**（是什么 / 关键点 / 例子 / 相关）、**决策**（背景 / 选项 / 定了啥 / 为什么 / 复盘钩子）、**复盘**（目标 / 结果 / 差距 / 归因 / 改进）、**读书笔记**（书 / 核心论点 / 我的收获 / 行动）、**项目**（目标 / 现状 / 下一步）。
