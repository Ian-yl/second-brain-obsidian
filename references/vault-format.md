# Vault 写入格式（agent-native 权威规范）

> agent 直接读写本 vault 就靠这份规范——严格照它写，产出稳定一致。
> `<vault>` = 库根（建库时定）；所有日期用**本地当天** `YYYY-MM-DD`。
> 结构理念：**四层知识流做一级目录、六大工作模式按需做中文子目录、行业只做映射页**（完整设计见 `docs/Obsidian-六大工作模式模板指南.md`）。
> **安全**：明文密钥 / token / 密码 **绝不写进 vault**——脱敏成「（已隐去）」再存。

## 目录布局
```
<vault>/
  00-Home.md              # 总入口索引（建库就建）
  用户画像.md              # 人格画像（单文件·留根）
  CLAUDE.md / AGENTS.md    # 画像摘要（两份同内容）；进 vault 原生自动读（Claude 读 CLAUDE.md，Codex/Hermes 读 AGENTS.md）
  00-Inbox/               # 收集箱：判不出归属的先放这
  10-Inputs/              # 输入·素材与来源
  20-Process/             # 加工·思考与方法
  30-Outputs/             # 产出·成果与决策
  40-Feedback/            # 反馈·复盘与洞察
  50-MOCs/                # 六大工作模式 MOC（建库就建 6 张）
  60-Domains/             # 行业映射页（按需建）
  70-Assets/              # 图片 / PDF / 附件 / 素材
  90-Archive/             # 完结 / 废弃 / 历史
  _System/                # Templates/  Dashboards/  Attachments/
```
- **10/20/30/40 四层下，按需建 6 个中文工作模式子目录**：`研究分析` `项目交付` `客户个案` `内容生产` `流程运营` `学习成长`。
- **建库只建**：`00-Home.md` + 上面所有一级目录（含空的 60/70/90/_System）+ `50-MOCs/` 里 6 张 MOC + 空骨架 `用户画像.md` / `CLAUDE.md` / `AGENTS.md`。
- **按需建**：四层下的二级模式子目录、`60-Domains/<domain>.md`。**不要一上来建 24 个空子目录**（造成填充压力）。

## 六大工作模式（目录用中文名；英文 ID 只在 frontmatter / 脚本里用）
> 这张表**既是模式定义、也是判位关键词**——§自动判位 直接对照它，别再单列第二张表。
| 中文目录名 | 英文 ID | 关键词线索（对象 / 动作 / 产出） |
|---|---|---|
| 研究分析 | `research-analysis` | 资料 / 文献 / 证据 / 调研 / 数据 / 结论 / 洞察 |
| 项目交付 | `project-delivery` | 项目 / 需求 / 里程碑 / 方案 / 决策 / 交付 / 发布 |
| 客户个案 | `case-management` | 客户 / 病例 / 案件 / 候选人 / 学生 / 房源 / 联系人 |
| 内容生产 | `content-production` | 选题 / 素材 / 草稿 / 文章 / 视频 / 课程 / 发布 |
| 流程运营 | `operations` | SOP / 清单 / 告警 / 事故 / 审计 / 异常 / 改进 |
| 学习成长 | `learning-growth` | 课程 / 概念 / 练习 / 日记 / 习惯 / 复习 / 反思 |

## 自动判位（写每条知识前，按这个顺序判）
1. **知识层**：它是输入(Inputs) / 加工(Process) / 输出(Outputs) / 反馈(Feedback)？拿不准默认 `Inputs`。
2. **工作模式**：按「**对象 + 动作 + 产出**」对照**上面「六大工作模式」表的关键词线索**判——

   - **冲突看当前目的**（同一份用户访谈，**层 + 模式都随目的变**：为发现需求→`10-Inputs/研究分析/`；为推进项目→`20-Process/项目交付/`；为维护客户→`20-Process/客户个案/`；为写文章→`10-Inputs/内容生产/`）。
   - **判不出 → 放 `00-Inbox/`**（frontmatter `mode: inbox`），稍后再归类，**别硬塞**。
3. **行业 domain**（可选）：标进 frontmatter，**不进目录**。
- 确定层+模式后，**目标二级目录不存在就自动创建**：`<层一级目录>/<中文模式>/`。
- **落位示范**（照着判）：

  | 内容 | 层 | 模式 | 路径 | domain |
  |---|---|---|---|---|
  | 用户反馈原始记录 | Inputs | 研究分析 | `10-Inputs/研究分析/` | product |
  | 病例学习笔记 | Process | 客户个案 | `20-Process/客户个案/` | healthcare |
  | 已发布文章 | Outputs | 内容生产 | `30-Outputs/内容生产/` | writing |
  | 事故复盘 | Feedback | 流程运营 | `40-Feedback/流程运营/` | sre |
  | 课程笔记 | Process | 学习成长 | `20-Process/学习成长/` | student |

- **边界（别做）**：① 不批量预建 24 个二级目录（用到才建）② 不按行业建一级目录（行业只进 `60-Domains/` 映射页）③ 目录名用中文（英文 ID 只在 frontmatter/脚本里）④ `60-Domains/` 只放映射、不存正文 ⑤ 判位时不生成正文内容模板。

## 1. `用户画像.md`
frontmatter + **固定 8 段（顺序不可变，空段也保留标题）**：
```
---
type: user-profile
updated: <YYYY-MM-DD>
framework_version: 1
tags: [<标签>, ...]            # 内联列表，形如 性格/完美主义、风格/简洁直接
---

## 概览
<一两句总览；没有就写「（待补充）」>

## 性格特质
- <一句话特征>

## 价值观 / 做事原则
## 沟通 / 说话风格
## 思维 / 做事方式
## 行为处事 / 待人接物
## 偏好与禁忌

## 变更日志
- <YYYY-MM-DD> · [<维度>] <内容> · 来源 <source>
```
- **6 个维度名固定、别改字**：`性格特质` / `价值观 / 做事原则` / `沟通 / 说话风格` / `思维 / 做事方式` / `行为处事 / 待人接物` / `偏好与禁忌`。每段 0+ 条 `- ` 要点。
- **合并规则**：
  - 加一条特征 → 进对应维度段，**先去重**（跟已有要点意思重复就别加）再 append `- <内容>`。
  - 修正既有 → 找意思相近的那条**替换**，找不到就 append。
  - 每加/改一条 → `变更日志` append 一行（日期·`[维度]`·内容·`来源 <source>`），日志**只留最近 50 条**。
  - 新标签并进 frontmatter `tags`（去重）；动过就把 `updated` 设成当天。

## 2. 知识笔记 `<层一级目录>/<中文模式>/<YYYY-MM-DD> - <标题>.md`
```
---
type: knowledge
layer: <Inputs|Process|Outputs|Feedback>
mode: <research-analysis|project-delivery|case-management|content-production|operations|learning-growth|inbox>
domain: <行业/角色，可空>
created: <YYYY-MM-DD>
updated: <YYYY-MM-DD>
aliases: [<别名>, ...]
tags: [<领域/xx>, <类型/xx>, 层/<层>, 模式/<中文模式>]   # tags 必含 层/<层> 和 模式/<中文模式>
sources: [<source>, ...]
---

# <标题>

<正文（markdown 结构：`## 小标题` 分段 / `- ` 列表 / `**强调**` / `>` 引用 / 代码块，别写成一坨纯文本）>

相关：[[<相对路径不带.md>|<显示标题>]] [[...]]
```
- **正文 = 结构化 markdown**（**别写成一坨纯文本**）：用 `## 小标题` 分段、`- ` / `1.` 列表、`**强调**`、`>` 引用、` ``` ` 代码块、`[[路径|别名]]` 链接组织；有对应类型就套 `note-templates.md`（概念 / 决策 / 复盘 / 读书笔记 / 项目）。
- **路径** = `<层一级目录>/<中文工作模式>/`（如 `10-Inputs/研究分析/`）；目录不存在**自动建**；模式判不出 → 放 `00-Inbox/`（`mode: inbox`、不进二级目录、tags 里用 `模式/inbox`）。
- **文件名** = `<YYYY-MM-DD> - <标题>.md`（标题去掉 `\ / : * ? " < > |`、≤60 字；可加短前缀如 `2026-06-30 - product - 用户访谈.md`）。**别用** `最终版.md` / `新建文档.md` / `未命名.md` / `杂项.md` 这类无意义名。
- **去重/合并·按先例落位**：写前先搜——① 该层+该模式目录内按**标题/别名**找；② **跨层/模式**也扫**同主题/同实体**。**有 → 合并进那一篇**（`tags`/`aliases`/`sources` 取并集、`updated`=今天、正文**不重复段落**才 append），并**沿用它已有的位置**（同一主题/实体别另起第二处，防判位漂移）；无 → 才新建。
- **🔗 链接生成规范（重要 · 防断链）**：链接到**知识笔记**（文件名带 `<日期> - ` 前缀）**必须用「相对路径别名」格式** `[[<相对路径不带.md>|<显示标题>]]`——例：`[[10-Inputs/学习成长/2026-07-01 - 浮生物语|浮生物语]]`。**路径直接解析、不靠 alias**（裸 `[[标题]]` 会因文件名带日期前缀而断链）。MOC / `00-Home` / `相关:` 里凡是指向知识笔记的链接**一律用这个格式**。
  - **例外 · 固定名文件用短链即可**：`用户画像`、`00-Home`、`MOC - <模式>`、`60-Domains/<domain>`——名字固定、无日期前缀、basename 即链接名 → `[[用户画像]]` / `[[MOC - 学习成长]]` / `[[product]]`。
- **`aliases`**：只做**搜索 / 显示辅助**（可放标题、简称、旧名），**不作为链接解析依据**——链接靠路径，不靠 alias。
- **`相关：` 行**：只有有 links 时才加；链接一律用**路径别名格式** `[[<相对路径不带.md>|<显示标题>]]`。
- **`sources` 标来源**：`用户明说`（事实，可自动写）/ `推断·待确认`（你演绎出来的，别当事实——宜先问用户再升级为事实）。

## frontmatter 语法
- 标量：`key: value`；列表：`key: [a, b, c]`。值里含 `, [ ] : #` 或首尾空格的，用双引号包：`"v"`。
