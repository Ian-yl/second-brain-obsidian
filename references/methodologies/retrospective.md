# 复盘法

## 是什么
做完一件事后**系统回看**，把经验变成下次能用的改进——不是"感觉一下"，而是走固定几步：目标 → 结果 → 差距 → 归因 → 改进（≈ PDCA / 军队 AAR）。

## 怎么用（步骤）
1. **回顾目标**：当初想达成什么？
2. **对照结果**：实际发生了什么？
3. **找差距**：哪超预期、哪没达到？
4. **归因**：为什么？（分"可控 / 不可控"、"能力 / 运气"）
5. **提炼改进**：下次具体怎么做？写成可执行的一条。

## 什么时候用
项目 / 事故 / 一段时间（周 / 月）结束后；任何想"别再犯同样错"的时候。

## 一个例子
上线延期复盘：目标=周五上线，结果=拖到周一，差距=晚 1 天，归因=大文件没压测(可控)，改进=下次上线前必做大文件压测 → 写进 `40-Feedback`。
